## 1. 시스템 개요

### 핵심 컨셉

```
┌─────────────────────────────────────────────────────────────┐
│                      My AI Company                          │
│                                                             │
│   📁 Input (요청)  →  🤖 AI 에이전트  →  📁 Output (결과)   │
│                                                             │
│   • 총괄 (Orchestrator): 계획 수립                          │
│   • 실무 (Worker): 작업 실행                                │
│   • 검증 (Validator): 품질 승인                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 3. 팀별 사용법

### 01_Strategy_Office (전략기획실)

**용도:** 중요한 의사결정 전 다각도 검토가 필요할 때

**Input 예시:**
```markdown
# 전략 안건 요청서
## 안건명: [검토가 필요한 안건]
## 배경: [현재 상황 설명]
## 검토 요청 사항
1. [질문 1]
2. [질문 2]
```

**Output:** 전략 보고서 + 실행 로드맵 + Executive Summary

**특징:** 4인(낙관/비관/현실/혁신) 토론 후 합의 도출

---

### 02_Marketing_Team (마케팅팀)

**용도:** 하나의 콘텐츠를 여러 채널용으로 변환할 때

**Output 예시:**
- OSMU 콘텐츠 패키지 (5채널)
- 발행용 파일 (복사해서 바로 게시)
- IR 피칭 대본, 카드뉴스

---

### 03_Content_Team (콘텐츠팀)

**용도:** 블로그 글, 아티클 등 롱폼 콘텐츠가 필요할 때

**Output:** SEO 최적화 블로그 글 (2,000-3,000자)

---

### 04_Visual_Team (비주얼팀)

**용도:** 썸네일, 배너, 이미지가 필요할 때

**Output:** Midjourney/DALL-E 프롬프트 + 디자인 가이드

---

### 05_Ecommerce_Team (이커머스팀)

**용도:** 상품 상세페이지 기획이 필요할 때

---

### 06_Finance_Team (재무팀)

**용도:** 매출/비용 데이터 분석이 필요할 때

---

### 07_BizDev_Team (사업개발팀)

**용도:** 정부사업/입찰/클라이언트 제안서가 필요할 때

**Output:** 사업계획서 + 윈닝 전략 + QA 검증 보고서

---

### 08_Education_Team (교육팀)

**용도:** 강의 자료 풀세트가 필요할 때

---

### 09_Brainstorming_Team (브레인스토밍팀)

## 4. Quick Start

### Step 1: 팀 선택
```
필요한 업무에 맞는 팀 폴더로 이동
예: 콘텐츠가 필요하면 → 03_Content_Team
```

### Step 2: Input 파일 작성
```
Input 폴더에 요청 파일 생성
파일명: [유형]_[날짜]_[주제].md
예: article_20260202_AI자동화.md
```

### Step 3: 에이전트 실행
```
해당 팀의 agent_config.md를 시스템 프롬프트로 사용
Input 파일 내용과 함께 AI에게 작업 요청
```

### Step 4: Output 확인
```
결과물이 Output 폴더에 저장됨
QA 검증 결과 확인 후 사용
```

## 5. 파이프라인 (팀 간 연결)

### 예시: 사업개발팀 → 마케팅팀 파이프라인

```
╔═══════════════════════════════════════════════════════════════╗
║                    PIPELINE WORKFLOW                           ║
╠═══════════════════════════════════════════════════════════════╣
║                                                                ║
║  STEP 1: 사업개발팀                                            ║
║  │  Input: 공고문/RFP                                          ║
║  │  Output: 사업계획서                                         ║
║  ▼                                                             ║
║  STEP 2: 품질검증                                              ║
║  │  누락 항목 체크                                             ║
║  │  Output: QA 검증보고서 (승인/반려)                          ║
║  ▼                                                             ║
║  STEP 3: 파일 이관                                             ║
║  │  07_BizDev/Output → 02_Marketing/Input                     ║
║  ▼                                                             ║
║  STEP 4: 마케팅팀                                              ║
║     Input: 사업계획서                                          ║
║     Output: IR 피칭대본 + 카드뉴스                             ║
║                                                                ║
╚═══════════════════════════════════════════════════════════════╝
```

### 실행 명령 예시
```
"사업개발팀의 사업계획서를 마케팅팀으로 연결해서
IR 피칭 대본과 카드뉴스를 만들어줘"
```

## 6. 병렬 실행

### 여러 팀 동시 실행

```
╔═══════════════════════════════════════════════════════════════╗
║                    PARALLEL EXECUTION                          ║
╠═══════════════════════════════════════════════════════════════╣
║                                                                ║
║  🔄 Agent 1: 콘텐츠팀 ─────────── 블로그 글 작성              ║
║  🔄 Agent 2: 마케팅팀 ─────────── 카드뉴스 제작               ║
║  🔄 Agent 3: 비주얼팀 ─────────── 이미지 프롬프트             ║
║                                                                ║
║  → 3개 팀이 동시에 작업 수행                                   ║
║  → 완료되면 각각 Output 폴더에 저장                           ║
║                                                                ║
╚═══════════════════════════════════════════════════════════════╝
```

### 실행 명령 예시
```
"콘텐츠팀, 마케팅팀, 비주얼팀에서
'AI 1인 기업' 주제로 병렬 작업 실행해줘"
```

## 9. FAQ

### Q: 어떤 AI 도구를 사용해야 하나요?
**A:** Claude 또는 ChatGPT 권장. agent_config.md를 시스템 프롬프트로 사용하고, Input 파일 내용을 전달하면 됩니다.

### Q: 결과물 품질이 낮으면 어떻게 하나요?
**A:**
1. Input을 더 구체적으로 작성
2. agent_config.md의 품질 기준 강화
3. QA 검증 단계에서 피드백 후 재작업 요청

### Q: 팀을 추가하거나 수정할 수 있나요?
**A:** 네. 새 폴더 생성 후 agent_config.md를 작성하면 됩니다.

### Q: 여러 팀을 한 번에 사용할 수 있나요?
**A:** 네. 병렬 실행으로 동시에 작업하거나, 파이프라인으로 순차 연결할 수 있습니다.

### Q: Output을 바로 사용해도 되나요?
**A:** 반드시 검토 후 사용하세요. AI는 초안을 생성하고, 최종 판단은 사람이 해야 합니다.
