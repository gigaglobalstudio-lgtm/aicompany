## 1. 시스템 개요

### 핵심 컨셉

```
┌─────────────────────────────────────────────────────────────┐
│                      My AI Company                          │
│                                                             │
│   📁 Input (요청)  →  🤖 AI 에이전트  →  📁 Output (결과)   │
│                                                             │
│   • 총괄 (Orchestrator): 계획 수립                          │
│   • 실무 (Worker): 작업 실행                                │
│   • 검증 (Validator): 품질 승인                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 10개 팀 (부서)

| # | 팀명 | 역할 | 핵심 기능 |
|---|------|------|----------|

| 06 | **재무팀** | 데이터 분석 | 재무 리포트 + 시각화 |
| 07 | **사업개발팀** | 제안서 | 정부사업/입찰 대응 |

| 09 | **브레인스토밍팀** | 아이디어 | 변증법 토론 → 실행안 |
| 10 | **경영지원팀** | 백오피스 | 문서, 이메일, 회의록 |

## 3. 팀별 사용법

### 01_Strategy_Office (전략기획실)

**용도:** 중요한 의사결정 전 다각도 검토가 필요할 때

**Input 예시:**
```markdown
# 전략 안건 요청서

## 안건명
[검토가 필요한 안건]

## 배경
[현재 상황 설명]

## 검토 요청 사항
1. [질문 1]
2. [질문 2]

## 희망 산출물
- 전략 보고서
- Go/No-Go 권고안
```

**Output:** 전략 보고서 + 실행 로드맵 + Executive Summary

**특징:** 4인(낙관/비관/현실/혁신) 토론 후 합의 도출

---

### 02_Marketing_Team (마케팅팀)

**용도:** 하나의 콘텐츠를 여러 채널용으로 변환할 때

**Input 예시:**
```markdown
# 마케팅 콘텐츠 요청

## 원본 콘텐츠
[유튜브 링크 / 블로그 글 / 주제]

## 핵심 메시지
[전달하고 싶은 핵심]

## 타겟 오디언스
[대상 정의]

## 요청 채널
1. 블로그 (SEO)
2. 인스타그램
3. 뉴스레터
4. 트위터
5. 링크드인
```

**Output:** 5채널 OSMU 콘텐츠 패키지

## 콘텐츠 유형
[롱폼 아티클 / 케이스 스터디 / How-To 가이드]

## 주제
[주제]

## 목적
[인지 / 교육 / 전환]

## 타겟 독자
[독자 정의]

## 요청 분량
[2,000-3,000자 등]
```

**Output:** 완성된 아티클 (SEO 최적화, CTA 포함)

## 4. Quick Start

### Step 1: 팀 선택
```
필요한 업무에 맞는 팀 폴더로 이동
예: 콘텐츠가 필요하면 → 03_Content_Team
```

### Step 2: Input 파일 작성
```
Input 폴더에 요청 파일 생성
파일명: [유형]_[날짜]_[주제].md
예: article_20260202_AI자동화.md
```

### Step 3: 에이전트 실행
```
해당 팀의 agent_config.md를 시스템 프롬프트로 사용
Input 파일 내용과 함께 AI에게 작업 요청
```

### Step 4: Output 확인
```
결과물이 Output 폴더에 저장됨
QA 검증 결과 확인 후 사용
```

## 5. Input/Output 규칙

### Input 파일 명명 규칙
```
[유형]_[날짜]_[주제].md

예시:
agenda_20260202_신사업검토.md      (전략기획실)
request_20260202_AI콘텐츠.md       (마케팅팀)
product_20260202_컨설팅패키지.md   (이커머스팀)
```

### Output 파일 명명 규칙
```
[산출물유형]_[주제]_[날짜].md

예시:
strategy_report_20260202.md        (전략 보고서)
osmu_콘텐츠_20260202.md            (OSMU 패키지)
financial_report_20260202.md       (재무 리포트)
```

### 필수 포함 요소

**Input:**
- [ ] 요청 배경/목적
- [ ] 구체적 요청 사항
- [ ] 제약 조건 (있다면)
- [ ] 희망 산출물 형태

**Output:**
- [ ] 요청에 대한 답변
- [ ] QA 검증 결과
- [ ] 팀/작성일 명시

## 6. 고급 활용법

### 병렬 실행
```
여러 터미널을 열어 각 팀을 동시에 가동

Terminal 1: 전략기획실 (중요 안건 검토)
Terminal 2: 마케팅팀 (콘텐츠 변환)
Terminal 3: 콘텐츠팀 (블로그 글 작성)
```

### 팀 간 연계
```
01_전략기획실 → 방향 결정
      ↓
03_콘텐츠팀 → 콘텐츠 작성
      ↓
02_마케팅팀 → 채널별 변환
      ↓
04_비주얼팀 → 이미지 기획
```

### 프로젝트 폴더 구조 (대형 프로젝트)
```
My_AI_Company/
└── Projects/
    └── 2026_신규서비스_런칭/

        ├── 02_콘텐츠 (콘텐츠팀 Output)
        ├── 03_마케팅 (마케팅팀 Output)

## 7. FAQ

### Q: 어떤 AI 도구를 사용해야 하나요?
**A:** Claude 또는 ChatGPT 권장. agent_config.md를 시스템 프롬프트로 사용하고, Input 파일 내용을 전달하면 됩니다.

### Q: 결과물 품질이 낮으면 어떻게 하나요?
**A:**
1. Input을 더 구체적으로 작성
2. agent_config.md의 품질 기준 강화
3. QA 검증 단계에서 피드백 후 재작업 요청

### Q: 팀을 추가하거나 수정할 수 있나요?
**A:** 네. 새 폴더 생성 후 agent_config.md를 작성하면 됩니다. 기존 팀의 구조를 참고하세요.

### Q: 여러 팀을 한 번에 사용할 수 있나요?
**A:** 네. 터미널을 여러 개 열어 병렬 실행하거나, 한 세션에서 순차적으로 요청할 수 있습니다.

### Q: Output을 바로 사용해도 되나요?
**A:** 반드시 검토 후 사용하세요. AI는 초안을 생성하고, 최종 판단은 사람이 해야 합니다.
