## 도입부

매일 같은 이메일을 쓰고, 같은 보고서를 만들고, 같은 데이터를 정리하고 있지 않으신가요?

한 조사에 따르면, 직장인은 하루 평균 **2.5시간**을 반복적인 업무에 소비합니다. 연간으로 환산하면 **650시간**, 약 **81일**입니다.

만약 이 시간의 절반만 줄일 수 있다면 어떨까요?

AI 업무 자동화는 더 이상 대기업만의 이야기가 아닙니다. 이 글에서는 누구나 따라 할 수 있는 **7단계 프레임워크**를 소개합니다. 코딩 지식 없이, ChatGPT나 Claude 같은 도구만으로 시작할 수 있습니다.

## Step 1: 반복 업무 목록 작성하기

### 왜 중요한가?
자동화의 첫 단계는 **무엇을 자동화할지** 파악하는 것입니다. 의외로 많은 사람들이 자신이 어떤 반복 업무를 하는지 인식하지 못합니다.

### 어떻게 하나?

**1주일 동안 업무 기록하기:**
| 시간 | 업무 | 반복 주기 | 소요 시간 |
|------|------|----------|----------|
| 09:00 | 이메일 확인/답장 | 매일 | 30분 |
| 10:00 | 주간 보고서 작성 | 주 1회 | 2시간 |
| 14:00 | SNS 콘텐츠 작성 | 주 3회 | 1시간 |

**자동화 후보 기준:**
- ✅ 주기적으로 반복됨 (매일/매주)
- ✅ 명확한 입력과 출력이 있음
- ✅ 창의성보다 정확성이 중요함
- ✅ 현재 많은 시간을 소비함

### 실전 팁
> "자동화할 가치가 있는가?"를 판단하는 공식:
> **연간 절약 시간 = 주간 소요 시간 × 52주 × 0.7 (자동화율)**
>
> 예: 주 2시간 업무 → 연간 73시간 절약

## Step 2: 자동화 우선순위 결정하기

### 왜 중요한가?
모든 업무를 한 번에 자동화할 수는 없습니다. **임팩트가 큰 것**부터 시작해야 동기 부여도 되고, 효과도 빠르게 체감합니다.

### 우선순위 매트릭스

```
높음 │  ②         │  ① 먼저     │
     │  나중에     │  시작!      │
임팩 ├────────────┼────────────┤
트   │  ④         │  ③         │
     │  안 함     │  시간 나면   │
낮음 └────────────┴────────────┘
        어려움          쉬움
              구현 난이도
```

### 실전 예시

| 업무 | 임팩트 | 난이도 | 우선순위 |
|------|--------|--------|----------|
| 이메일 템플릿 자동화 | 중 | 쉬움 | ★★★★☆ |
| 보고서 초안 생성 | 상 | 중간 | ★★★★★ |
| SNS 콘텐츠 변환 | 상 | 중간 | ★★★★★ |
| 회의록 자동 요약 | 중 | 쉬움 | ★★★★☆ |
| 데이터 분석 자동화 | 상 | 어려움 | ★★★☆☆ |

## Step 3: 프로세스 문서화하기

### 왜 중요한가?
AI에게 일을 시키려면, **당신이 어떻게 일하는지**를 명확히 알려줘야 합니다. 이것이 바로 '프로세스 문서화'입니다.

### 문서화 템플릿

```markdown
## 업무명: [예: 주간 보고서 작성]

### 입력 (Input)
- 필요한 데이터: [예: 주간 매출 데이터, 주요 이슈 목록]
- 형식: [예: 엑셀 파일, 텍스트]

### 처리 과정 (Process)
1. 매출 데이터에서 전주 대비 변화율 계산
2. 상위 5개 이슈 선별
3. 각 이슈별 원인과 대응 방안 작성
4. 임원 보고용 요약문 작성

### 출력 (Output)
- 형식: [예: 워드 문서, 2페이지 이내]
- 포함 요소: 요약, 상세 분석, 액션 아이템
```

### 실전 팁
> 동료에게 설명하듯이 적으세요.
> "아, 그건 그냥 알아서 해"라는 부분이 없어야 합니다.

## Step 4: AI 도구 선택하기

### 주요 도구 비교

| 도구 | 강점 | 약점 | 추천 용도 |
|------|------|------|----------|
| **ChatGPT** | 범용성, 플러그인 | 긴 문서 처리 한계 | 이메일, 짧은 글 |
| **Claude** | 긴 문서, 분석 | 이미지 생성 불가 | 보고서, 분석 |

| **Notion AI** | 문서 내 통합 | 범용성 낮음 | 노션 사용자 |

### 선택 기준
1. **이미 쓰는 도구**가 있다면 그것부터
2. **무료 플랜**으로 테스트 먼저
3. **복잡한 도구**보다 **간단한 것**부터

## Step 5: 프롬프트 설계하기

### 왜 중요한가?
AI의 성능은 **프롬프트 품질**에 80% 좌우됩니다. 잘 설계된 프롬프트는 일관되고 고품질의 결과를 만듭니다.

### 프롬프트 구조 (ROSE 프레임워크)

```
[R] Role (역할)
"너는 10년차 마케팅 전문가야."

[O] Objective (목표)
"블로그 글의 핵심을 3줄로 요약해줘."

[S] Specification (세부사항)
"각 줄은 50자 이내, 불릿포인트 형식으로."

[E] Example (예시)
"예시: • 핵심1 • 핵심2 • 핵심3"
```

### 실전 예시: 이메일 답장 자동화

```
Role: 너는 친절하고 전문적인 고객 서비스 담당자야.

Task: 아래 고객 문의에 대한 답장을 작성해줘.

Rules:
- 먼저 문의에 감사 표현
- 핵심 답변을 명확히
- 추가 도움 제안으로 마무리
- 200자 이내
- 존댓말 사용

고객 문의:
[여기에 문의 내용 붙여넣기]
```

## Step 6: 테스트 및 개선하기

### 왜 중요한가?
처음부터 완벽한 자동화는 없습니다. **반복 테스트**를 통해 품질을 높여야 합니다.

### 테스트 체크리스트

- [ ] 결과물이 기대한 형식인가?
- [ ] 내용이 정확한가?
- [ ] 톤앤매너가 적절한가?
- [ ] 예외 상황에서도 작동하는가?
- [ ] 수동 작업보다 실제로 빠른가?

### 개선 사이클

```
프롬프트 작성 → 테스트 실행 → 결과 검토
      ↑                              ↓
      └──────── 프롬프트 수정 ←──────┘
```

### 실전 팁
> 10번 테스트해서 8번 이상 만족스러우면 실전 투입하세요.
> 100% 완벽을 기다리면 영원히 시작 못 합니다.

## 자주 묻는 질문 (FAQ)

### Q: AI가 만든 결과물을 그대로 써도 되나요?
A: **검토는 필수**입니다. AI는 초안을 만들고, 사람이 검토/수정합니다. 이것만으로도 시간의 70%는 절약됩니다.

### Q: 어떤 업무는 자동화하면 안 되나요?
A: **관계 구축**, **창의적 의사결정**, **민감한 커뮤니케이션**은 사람이 직접 해야 합니다. 자동화는 '대체'가 아닌 '보조'입니다.

### Q: 비용이 많이 드나요?
A: ChatGPT Plus는 월 $20, Claude Pro는 월 $20 수준입니다. 절약되는 시간 가치를 생각하면 충분히 ROI가 나옵니다.

### Q: 코딩을 몰라도 되나요?
A: **네.** 이 가이드의 모든 내용은 코딩 없이 가능합니다. 프롬프트 작성 능력만 있으면 됩니다.
