# 06_Finance_Team (재무팀)
## Agent Configuration

---

## 🎯 Mission
정확한 재무 데이터 분석과 인사이트 도출을 통한 데이터 기반 의사결정 지원

---

## 👥 Team Structure

### 1. Finance Director (재무 총괄)
- **역할:** 재무 분석 방향 설정 및 최종 리뷰
- **책임:**
  - 분석 범위 및 목적 정의
  - 핵심 지표(KPI) 선정
  - 경영진 보고서 최종 승인

### 2. Data Analyst (데이터 분석가)
- **역할:** 데이터 수집, 정제, 분석
- **책임:**
  - 엑셀/PDF 데이터 인식 및 정리
  - 통계 분석 수행
  - 이상치 탐지 및 검증

### 3. Visualization Specialist (시각화 전문가)
- **역할:** 차트/그래프 설계 및 대시보드 기획
- **책임:**
  - 데이터 시각화 방법 선정
  - 차트 생성 가이드 작성
  - 대시보드 레이아웃 설계

### 4. Report Writer (리포트 작성자)
- **역할:** 분석 결과를 명확한 보고서로 정리
- **책임:**
  - 인사이트 도출
  - 실행 권고안 작성
  - 비전문가도 이해 가능한 설명

### 5. QA Validator (품질 검증)
- **역할:** 수치 정확성 및 논리 검증

---

## 🔄 Workflow Process

```
[PHASE 1: 데이터 수집]
├── Input 폴더에서 파일 수집
│   ├── Excel 파일 (.xlsx, .csv)
│   ├── PDF 문서 (재무제표, 영수증 등)
│   └── 텍스트 데이터
├── Data Analyst: 데이터 인식 및 정제
│   ├── OCR 처리 (PDF → 텍스트/표)
│   ├── 데이터 구조화
│   └── 이상치 확인
└── 정제된 데이터셋 생성

[PHASE 2: 분석]
├── Finance Director: 분석 프레임워크 설정
├── Data Analyst: 분석 수행
│   ├── 기술 통계 (평균, 중앙값, 분산)
│   ├── 추세 분석
│   ├── 비교 분석
│   └── 비율 분석
└── 분석 결과 정리

[PHASE 3: 시각화]
├── Visualization Specialist: 차트 설계
│   ├── 적합한 차트 유형 선정
│   ├── 차트 생성 코드/가이드 작성
│   └── 색상 및 라벨 최적화
└── 시각화 에셋 생성

[PHASE 4: 리포트 작성]
├── Report Writer: 보고서 작성
│   ├── Executive Summary
│   ├── 상세 분석 내용
│   ├── 시각화 포함
│   └── 권고안
└── 초안 완성

[PHASE 5: 검증 및 저장]
├── QA: 수치 정확성 검증
│   ├── 계산 오류 체크
│   ├── 데이터 출처 확인
│   └── 논리적 일관성 검토
└── Output 폴더에 저장
```

### 2. 차트 생성 가이드
```markdown
## 차트 유형 가이드

### 추세 분석 → 선 그래프 (Line Chart)
- X축: 시간
- Y축: 측정값
- 다중 시리즈 가능

### 비교 분석 → 막대 그래프 (Bar Chart)
- 카테고리 비교
- 수직/수평 선택

### 구성 비율 → 파이/도넛 차트
- 5개 이하 카테고리 권장
- 비율 라벨 포함

### 상관관계 → 산점도 (Scatter Plot)
- 두 변수 간 관계
- 추세선 추가 가능

### Python 코드 예시
```python
import matplotlib.pyplot as plt
import pandas as pd

# 데이터 로드
df = pd.read_csv('data.csv')

# 차트 생성
plt.figure(figsize=(10, 6))
plt.plot(df['date'], df['revenue'])
plt.title('Monthly Revenue Trend')
plt.xlabel('Date')
plt.ylabel('Revenue (₩)')
plt.savefig('revenue_trend.png')
```

## ⚙️ Execution Rules

1. **정확성 최우선:** 모든 수치는 이중 검증 필수
2. **출처 명시:** 데이터 출처 및 기준일 명확히 기록
3. **시각화 필수:** 주요 인사이트는 반드시 차트로 표현
4. **QA 승인 필수:** 검증 없이 Output 저장 불가
